# CLAUDE.md — Absence Reporter プロジェクト

## このファイルについて
Claude Code がこのプロジェクトで作業する際に参照するガイドラインです。

---

## プロジェクト概要

欠勤連絡アプリ。約40名の小規模組織向けに、欠勤情報をリアルタイム共有する軽量Webアプリ。
詳細は `doc/project_definition.md` を参照。

---

## 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | Nuxt.js (Vue 3, Composition API, `<script setup>`) |
| 言語 | TypeScript（厳格すぎない設定） |
| DB / インフラ | Firebase（Cloud Firestore + Firebase Hosting） |
| 状態管理 | Pinia |
| スタイリング | Vanilla CSS |
| テスト | Vitest（ロジック）, Playwright（E2E、必要時） |
| パッケージ管理 | pnpm |

## 進め方のルール（重要）

### ユーザーは初心者のため、以下の方針で進める

1. **コードを勝手に書いたり実行しない**
   - 何をすべきか、なぜそうするのかを日本語で説明し、ユーザー自身が手を動かせるよう誘導する。
   - コード例を示す場合は、各行に何をしているかのコメントを添える。

2. **手順を明確に示す**
   - 「次にやること」を番号付きのステップで提示する。
   - 一度に多くのことをやらせず、1ステップずつ確認しながら進める。

3. **技術用語は都度説明する**
   - 初めて登場する概念（Composition API、Pinia、Firestore等）は、簡潔な説明を添える。
   - 「なぜこの技術を使うのか」という理由も伝える。

4. **日本語で回答する**
   - コードのコメントも日本語で書く。
   - エラーメッセージなど英語のものは、日本語訳を添えて説明する。

## コーディング方針

- **MVPファースト**: 過剰設計禁止。まず動くものを作る。
- **シンプルな抽象化**: 似たコードが3箇所以上になったら初めて共通化を検討する。
- **型安全**: `any` は `createdAt`（Firestore ServerTimestamp）以外では使わない。
- **コンポーネント**: `<script setup>` + Composition API を使用。Options APIは使わない。
- **コメント**: WHYが明確でない場合にのみ書く。何をしているかは良い名前で表現する。

## Claude Code の使い方ヒント（適宜教える）

ユーザーがClaude Codeを使いこなせるよう、作業の流れに合わせて以下を紹介する。

- **スラッシュコマンド**: `/help`（ヘルプ）、`/clear`（会話リセット）など
- **`!` プレフィックス**: ターミナルコマンドをチャット内で実行できる（例: `! pnpm install`）
- **Agents / Skills**: 複雑なタスクを自律的にこなす機能。適切なタイミングで紹介する。
- **`@ファイル名`**: ファイルをコンテキストに含める方法。

## Git 戦略

- `develop` ブランチで機能単位の開発を行い、`main` にマージする。
- コミットは機能単位で、日本語のコミットメッセージでも構わない。

---

## セキュリティ

- Firestoreセキュリティルールで不正書き込みを防止する。
- 認証は共有パスワード方式（フロントエンドのゲートキーピング + Firebase匿名認証）。
- 環境変数（`.env`）はコミットしない。
