## Claude Code とは？

**Claude Code**はAnthropicが開発したCLI（コマンドラインインターフェース）ツールです。ターミナルから直接AIアシスタントと対話しながらコードを書いたり、バグを修正したり、ファイルを操作したりできます。

### 通常のAIチャットとの違い

| 通常のAIチャット（例: ChatGPT） | Claude Code |
|---|---|
| コードをコピペして貼り付ける | プロジェクトのファイルを直接読み書きできる |
| 「こんなコードはどうですか？」 | 実際にファイルを作成・編集する |
| ターミナルコマンドを自分で実行 | コマンドを代わりに実行できる |
| セッションをまたいで文脈が失われる | メモリ機能で文脈を保持できる |

Claude Codeはターミナル上で動作し、**あなたのプロジェクトファイルに直接アクセス**して実際に作業を行います。

## 3. スラッシュコマンド一覧

Claude Codeには `/` で始まる組み込みコマンドがあります。

### よく使うコマンド

| コマンド | 説明 | 使いどき |
|---|---|---|
| `/help` | ヘルプを表示 | 使い方を忘れたとき |
| `/clear` | 会話履歴をリセット | 別のタスクを始めるとき |
| `/compact` | 会話を要約して圧縮 | 長い会話でコンテキストを節約したいとき |
| `/status` | 現在の状態を表示 | 設定・権限モードの確認 |
| `/review` | コードレビューを依頼 | PRを出す前のチェック |
| `/vim` | Vim編集モードを切り替え | Vimユーザー向け |
| `/fast` | 高速モードの切り替え | 短い応答でいいとき |

### 会話の管理

```
# 新しいトピックに移るとき（前の文脈をリセット）
> /clear

# 長い会話を続けながら要約（コンテキスト節約）
> /compact
```

## 4. スキル（/skill）一覧

スキルは「あらかじめ用意された高度な指示セット」です。よく使う作業をワンコマンドで実行できます。

### 利用可能なスキル

#### `/commit` - Gitコミットを作成

```
> /commit
```

- 変更内容を自動で分析してコミットメッセージを生成
- ステージング〜コミットまで一括実行
- コミットメッセージの品質が自動で担保される

#### `/simplify` - コードの品質改善

```
> /simplify
```

- 変更したコードを振り返り、重複・複雑さ・非効率を発見して修正
- リファクタリングをサポート

#### `/schedule` - 定期実行タスクの設定

```
> /schedule
```

- cronスケジュールでエージェントを定期実行する設定
- 例: 毎朝9時にコードチェックを自動実行

#### `/loop` - 定期インターバル実行

```
> /loop 5m /commit
# 5分ごとにcommitスキルを実行
```

- 一定間隔でコマンドを繰り返す
- デフォルトは10分間隔

## 5. 権限モード

Claude Codeには3つの権限モードがあります。`Shift+Tab` または `Alt+M` で切り替えられます。

| モード | 説明 | 使いどき |
|---|---|---|
| **Default** | 危険な操作は確認を求める | 通常の開発作業 |
| **Auto-approve** | すべての操作を自動承認 | 信頼できるタスクを素早く実行したいとき |
| **Manual** | すべての操作に確認が必要 | 慎重に作業したいとき |

> **初心者向けアドバイス**: デフォルトモードのまま使うのが安全です。Claude Codeが「このファイルを削除してよいですか？」「このコマンドを実行してよいですか？」と聞いてくれます。

## 6. CLAUDE.md：プロジェクトの指示書

### CLAUDE.md とは？

プロジェクトのルートディレクトリに置く、**Claude Codeへの永続的な指示ファイル**です。会話を始めるたびに自動で読み込まれます。

「このプロジェクトではこのルールで開発してほしい」という約束事を書いておく場所です。

### 配置場所とスコープ

```
/home/jam00/work/absence-reporter/
  CLAUDE.md          ← プロジェクト全体への指示（全会話で読まれる）
  app/
    CLAUDE.md        ← app/ 配下の作業時のみ読まれる（任意）
```

### このプロジェクトでの CLAUDE.md 例

```markdown
# Absence Reporter - Claude への指示

## プロジェクト概要
約40名の小規模組織向け欠勤連絡Webアプリ。

## 技術スタック
- Nuxt 3 (Vue 3, Composition API, <script setup>)
- TypeScript（strictはfalse）
- Firebase (Firestore + Hosting + 匿名認証)
- Pinia, Vanilla CSS, Vitest, Playwright, pnpm

## ディレクトリ構成
- app/components/ : UIコンポーネント
- app/pages/      : ページ
- app/stores/     : Pinia ストア
- app/services/   : Firebase/外部APIロジック
- app/types/      : 型定義
- app/utils/      : ヘルパー関数

## コーディングルール
- MVPを優先。過剰設計・不要な抽象化は禁止。
- コンポーネントは <script setup lang="ts"> を使用する。
- スタイルは Vanilla CSS のみ（CSSフレームワーク禁止）。
- コメントは必要なロジックにのみ追加する。
- 型定義は app/types/ に集約する。

## Git ルール
- develop ブランチで機能開発し、main にマージする。
- コミットメッセージは日本語でOK。
```

### CLAUDE.md に書くと効果的な内容

- プロジェクトの概要・目的
- 使用技術・バージョン
- ディレクトリ構成の説明
- コーディング規約（禁止事項含む）
- テストの方針
- デプロイ手順

## 7. hooks：自動化の設定

### hooks とは？

**特定のイベント（ツールの実行前後）に自動でコマンドを実行する仕組み**です。`~/.claude/settings.json` に設定します。

### 設定ファイルの場所

```
~/.claude/settings.json   ← ユーザーグローバル設定（全プロジェクト共通）
.claude/settings.json     ← プロジェクトローカル設定（このプロジェクトのみ）
```

### settings.json の基本構造

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "echo 'ファイルを書き込もうとしています'"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "cd app && pnpm lint:fix"
          }
        ]
      }
    ]
  }
}
```

### 実用的なhooksの例

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "cd /home/jam00/work/absence-reporter/app && pnpm lint:fix 2>/dev/null || true",
            "description": "ファイル変更後に自動でlintを実行"
          }
        ]
      }
    ]
  }
}
```

> **補足**: hooksは慣れてきたら設定すればOKです。最初は使わなくても問題ありません。

### 基本原則：具体的に書く

```
# ❌ 曖昧な指示
> フォームを作って

# ✅ 具体的な指示
> app/pages/entry.vue に欠勤登録フォームを作成して。
> 入力項目は name（テキスト）, reason（セレクト: 体調不良/私用/その他）,
> startDate（date）, endDate（date）, note（テキストエリア・任意）。
> バリデーションはname・reason・startDate・endDateを必須とする。
> Vanilla CSSでスタイリングし、モバイル対応にする。
```

### ファイルを指定する

```
> app/services/absence.ts を作成して、Firestoreから当日の欠勤一覧を
> リアルタイム取得する composable を書いて
```

### エラーを貼り付ける

```
> 以下のエラーを修正して：
> [ERROR] Cannot read properties of undefined (reading 'value')
>   at AbsenceForm.vue:23
```

### 段階的に依頼する

大きな機能は一度に全部頼まず、小さく分けて依頼するのがコツです：

```
# まず型定義から
> Absence型をapp/types/absence.tsに定義して

# 次にサービス層
> Firestoreへの保存ロジックをapp/services/absence.tsに書いて

# 最後にUI
> 上のロジックを使うフォームコンポーネントをapp/components/AbsenceForm.vueに作成して
```

### 確認・説明を求める

```
> app/stores/absence.ts の内容を説明して
> このコードで問題になりそうな点はある？
> Firestoreのセキュリティルールを見直して、問題があれば修正して
```

## 11. よくある失敗と対処法

### 「意図しないファイルが変更された」

Claude Codeは確認を求めてくれますが、万が一のために：
- 作業前に `git status` で現在の状態を確認
- 定期的に `/commit` でセーブポイントを作る
- 不安なら `git diff` で変更差分を確認

### 「会話が長くなって的外れな回答が増えた」

```
> /clear
```

会話履歴をリセットして、新しいコンテキストから始めましょう。CLAUDE.mdに書いた内容は残ります。

### 「複雑なタスクを一度に頼んで混乱した」

タスクを小さく分解して、一つずつ確実に進めましょう。

### 「英語で返答が来た」

```
> 日本語で回答してください。以後の会話は日本語でお願いします。
```

または CLAUDE.md に「日本語で回答すること」と書いておく。

## まとめ：Claude Codeを最大限活用するための7か条

1. **CLAUDE.md を最初に書く** — プロジェクトのルールを一度書けば毎回伝える必要がない
2. **具体的に指示する** — ファイル名・関数名・仕様を明示する
3. **小さく分けて依頼する** — 一度に大きな機能を頼まない
4. **`/commit` を頻繁に使う** — こまめにセーブポイントを作る
5. **`/clear` を活用する** — 別のタスクを始めるときはリセット
6. **エラーはそのまま貼り付ける** — コピペで「このエラーを修正して」でOK
7. **確認を求める** — 「このコードに問題はある？」と聞くのは有効
